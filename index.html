Part 0: Initial Discussion¶

Our main data set: 🏛 ⚖ Orleans Parish Magistrate Court Dockets 🏛 ⚖¶

We are using dockets from the Orleans Parish Sheriff’s Office. Specifically, we scraped information from the following site(s): https://www.opso.us/dcktmstr/555555.php?&domagn=586999

This link contains court dockets for all cases in the Magistrate Court of Orleans parish. Varying the number in (“domagn=[num]”) gives prior court dockets. The following important information is given about each case:

  • defendant
  • judge
  • crime committed
  • bond price
  • hearing date
  • zoom/in-person
  • number of court appearances

As discussed in Part 1a-ii, for Milestone Two we are only analyzing dockets with magistrate numbers 554,000 to 556,000. We have compiled the scraped dockets into two csv files (also discussed in Part 1a-ii), which can be downloaded at this link.

Unfortunately, the race of the defendant is not included in the Docket, and there are several challenges with trying to get the race for each individual. Instead, we have run each name through software that gives the probability of an individual with a certain name being of a certain race, using the package ethnicolr. The documentation and source can be found at this github site

From this predictive model we are able to show any disparities for the bail bond system across different races and judges/comissioners.

An additional data set: 👮 🚔 2021 NOPD Incident Reports 👮 🚔¶

We have also implemented some analysis on the New Orleans Police Department Incident Reports from 2021. This data can be downloaded with this link

This data includes all incident reports thus far in 2021 made by NOPD. Because we are unable to get all demographics from the court dockets scraping, we are supplementing our data analysis to include general information on race and charges in New Orleans, to give a better understanding of the New Orleans Magistrate Court. Before the final deadline of this project, we would also like to (if possible) find a way of linking incident reports in which the offender was arrested to a specific docket. This would give way to some very strong analysis.

A statement on accuracy¶

Our scraping process is producing accurate data, as it is coming straight from public court records from which even the New Orleans Magistrate Court relies on. It is also complete, as the court system must have well kept records. The police records have some flaws, as the data includes every incident recorded, and therefore are missing key data points. However, we are combatting this by drilling down on the data on which a charge was documented, or an arrest was made. At this point in the incident, there is more accurate information which validates the data. Finally, we are using Ethnicolr to predict race based on last name. This process definitely introduces the most uncertainty in our data. The model is trained using Census data. An issue with this approach is that people of color are systematically underrepresented in Census Data, and using a model to predict names from a group of over policed people of color is problematic. To combat these disparities, we have set a tolerance of over 75& certainty in the names associated with Hispanic and Black races. For names associated with white race, we have set a tolerance of over 95%, as this demographic is the most represented in Census Data.

Our hopes, dreams, and goals¶

From these data sources, we hope to analyze the New Orleans Policing and Justice Department, specifically by asking:

  1. The percentage of arrests by race
  2. The count for common charges by race
  3. Distribution of people Arrested for a crime by race
  4. What is the relationship between judge, crime, and bond. Do different judges tend to set different bond prices for same crimes?
  5. Using the ethnicolr model: Is there a noticeable difference for bond price across different races when the crime is the same?
  6. The number of court appearance for a certain crime

Additionally, we hope to explore how video conferencing and zoom court hearings have affected bail/bond. Although, we have not been able to extract this information yet from scraping we are excited to ask:

  • What are the effects, if any, of Zoom/video conferencing court hearings on bond price (versus in person hearings)? If there is a noticeable difference in bond prices on Zoom versus in person, does it vary by judge?

Part 1: Data Scraping¶

Part 1a: Docket Scraping¶

A large component of our project is getting the actual data into an organized, tidy data frame. We are scraping court dockets, and each individual docket is stored on its own page. Complicating the process is the fact that the HTML is pretty much just text (dang messy HTML!), so, rather than using BeautifulSoup's .find() or .find_all() method to find certain data, we have to write several regular expressions to extract certain information from the entirety of the docket text. The process of scraping many thousands of dockets (talked about in part 1a-ii) is also very slow!

Part 1a-i: Necessary functions for docket scraping¶

The functions in this section are helper functions for scraping each docket. They are:

1) text_extract

  • Scrapes the entries at the bottom of a docket and returns a DataFrame with a row for each entry, where columns are 'Mag Num', 'Date', 'Judge', and 'Text'

2) get_mag

  • Scrapes docket for magistrate number and returns it

3) get_defendant_name

  • Scrapes docket for defendant name and returns it as string

4) get_date_judge_chargestext

  • Scrapes docket for date, judge, charges, and text (this is the text that is passed into the text_extract function). Returns all

5) extract_charges_bond

  • Using the charges from above, charge and bond as strings

6) create_docket_df_entry

  • Using the return values from functions 2-5, creates a new DataFrame for one docket, one row per charge

def nth_occurence(main_s, sub_s, n):
  '''Given a main string main_s and a substring sub_s, find the n_th occurence of sub_s in main_s'''

  # this code is taken from the following link: https://www.geeksforgeeks.org/python-ways-to-find-nth-occurrence-of-substring-in-a-string/

  # Finding nth occurrence of substring
  val = -1
  for i in range(0, n):
      val = main_s.find(sub_s, val + 1)
        
  # Printing nth occurrence
  return val

def text_extract(s, mag_id):
  '''This extracts the text between fourth and fifth lines and creates a new entity in a second data frame with columns:
    1) mag id
    2) date
    3) text
    4) judge'''
  dates = list()
  date_starts = list()
  # regex for dates proceeding a new line; loop over all
  for match in re.finditer(r'\n(\d{2}\/\d{2}\/\d{4})\s{2,}', s):
    dates.append(match.group().strip())
    date_starts.append(match.start())

  entities = list()
  for i,d in enumerate(date_starts):
    if i != len(dates) - 1:
      entities.append(s[date_starts[i]:date_starts[i+1]])
    else:
      fifth_line = nth_occurence(s, "="*78, 2)
      entities.append(s[date_starts[i]:fifth_line])
  texts = list()
  judges = list()
  for e in entities:
    reg = re.search(r'\n\s+[A-Z]', e)

    text = e[reg.start():].strip('\n')
    texts.append(text)

    reg1 = re.search(r'\s+([A-Z]+)', e)
    judge = reg1.group(1)
    judges.append(judge)
  
  return pd.DataFrame({'Mag Num': mag_id, 'Date': dates, 'Judge': judges, 'Text': texts})

def get_date_judge_chargestext(text):
    """Getting the date and judge for the proceeding and extracting the charges text"""
    # It is always two lines after the 4th equal sign line
    equalsignline = 0
    charges = []
    for (count,line) in enumerate(text):
        if line == "=" * 78:
            equalsignline +=1
        if equalsignline == 2:
            charges.append(text[count])
        elif equalsignline == 4:
            #print(mag_num, text[count+2]
            date, judge = re.sub('\*','',text[count+2]).split()
            proceedings = '\n'.join(text[count:])
            break
    return date, judge, charges, proceedings

def extract_charges_bond(charges):
    """Takes "charges" from above and extracts count, code, bond, and charge"""
    cnt = []
    code = []
    bond = []
    charge = []
    for i, line in enumerate(charges[3:-1]):
        #print("i and line:",i,line)
        if 'BOND:' in line:
            s = line.split()
            bond_ind = s.index("BOND:")
            cnt.append(s[0])
            code.append(" ".join(s[1:bond_ind]))
            bond.append(s[bond_ind +1])
        elif defendant in line:
            pass
        else:
            s = line.split()
            charge.append(" ".join(s[:]))
    return cnt, code, bond, charge

def create_docket_df_entry(mag_num, defendant, date, bond, judge, cnt, code, charge):
    # Duplicating the data that is the same for each charge on docket    
    mag_nums = [mag_num] * len(bond)
    defendants = [defendant] * len(bond)
    dates = [date] *len(bond)
    judges = [judge] * len(bond)
    #Create a dataframe using the extracted data
    docket = pd.DataFrame({"Mag Num": mag_nums,"Defendant": defendants, "Judge": judges, "Count": cnt, "Code": code,"Charge": charge, "Bond": bond, "Date": dates})
    return docket

Part 1a-ii: Scraping, organizing, cleaning, and combining court dockets¶

In this section we call upon the functions in Part 1-a in order to scrape dockets in a certain range of magistrate numbers. We store the data in two different DataFrames:

1) dockets which stores general information about the docket (a row here corresponds to a charge for a defendant, meaning one docket often maps to multiple rows)

2) proceedings_df which stores the text entries in the proceedings section. The unifying key between these DataFrames is "Mag Num".

Since scraping multiple dockets is pretty slow, we organize this code so that we can invoke our scrape_dockets() function at any time. Each call of srape_dockets(start, end) creates two csv files, dockets.csv and proceedings.csv, for the DataFrames dockets and proceedings_df, respectively. Of note is the fact that we wrap each scrape attempt of one docket in a "try, except" block because there are several dockets which do not scrape properly for one reason or another (often this is because the case number is not on file).

For the analsyis that follows, we have scraped dockets with magistrate numbers from 554,000 to 564,000, 2000 dockets at a time. Thus, we had five of each dockets.csv and proceedings.csv, each with a numerical extension saved in our directory. We use our combined_scraped_dfs() function to read in all csv's of scraped dockets and combine them into two master data frames, all_dockets and all_proceedings. combined_scraped_dfs() is useful across many rounds of scraping and storing interim csv files because it only reads in and combines csv files in the directory that it has not already done. Also, it saves all_dockets and all_proceedings to their own master csv files and reads in and adds to those csv files if its not the first call of the function. This allows us to scrape additional dockets whenever and continuously append to two master csv files (and their respective DataFrames), with no risk of duplicate entries (should the same docket be scraped more than once) or losing previously-scraped dockets.

def scrape_dockets(start_mag_num, end_mag_num):
    '''Scrape dockets in range start_mag_num to end_mag_num, exclusive'''


    '''
    NOTE TO SELF (11/16/21): prior to 554000, there was a mistake in text_extract:
      - i was doing s.index(date) to search for all date entries but sometimes
        date occurs in text of proceedings text, so this was causing nonetype issues

    What this means: may need to redo all dockets scraped prior to 554000


    NOTE TO SELF: -5 of dockets.csv and proceedings.csv are the beginning of 
    the file extensions beginning after fixing above
    '''
    # The prefix of every url we are scraping from
    base_url = 'https://www.opso.us/dcktmstr/555555.php?&domagn='


    # A list corresponding to the magistrate numbers of each docket
    l = list(range(start_mag_num, end_mag_num))

    soups = []

    dockets = pd.DataFrame()
    proceedings_df = pd.DataFrame()

    for num in l:
        try:
            r = requests.get(base_url + str(num))
            root = BeautifulSoup( r.content )
            soup = str(root.find('pre'))
            soup.__repr__()
            text = soup.split("\n")
            
            mag_num = get_mag(soup)
            defendant = get_defendant_name(text)
            date, judge, charges, proceedings = get_date_judge_chargestext(text)
            cnt, code, bond, charge = extract_charges_bond(charges)
            
            dockets = dockets.append(create_docket_df_entry(mag_num, defendant, date, bond, judge, cnt, code, charge), ignore_index = True)
            
            proceedings_df = proceedings_df.append(text_extract(proceedings, mag_num), ignore_index=True)
            time.sleep(1)

        except Exception as e:
            logger.error('Failed to upload to ftp: '+ str(e))
            print("Docket number {} is not working".format(str(num)))

    ## Initial cleaning of dataframe
    # Removes "," from the bond value and typecasts bond to numeric


    dockets["Bond"] = dockets["Bond"].str.replace(',','')
    dockets["Bond"] = pd.to_numeric(dockets["Bond"])


    dockets.to_csv("dockets.csv")
    proceedings_df.to_csv("proceedings.csv")

def combined_scraped_dfs():
    '''combined multiple csvs resulting from scraping into one'''

    # get list of files in 
    name_files = os.listdir(os.path.join("..", "milestone2", "dockets"))


    # initialize master data frames that we will combine smaller csvs into
    if 'all_dockets.csv' in name_files:
      # if we've already run this script before, read in the master csv
      all_dockets_df = pd.read_csv("all_dockets.csv")
      all_proceedings_df = pd.read_csv("all_proceedings.csv")
    else:
      # otherwise, initialize new master data frames
      all_dockets_df = pd.DataFrame()
      all_proceedings_df = pd.DataFrame()



    # text file storing dockets that we've already added to master csvs all_dockets and all_proceedings
    file = open("already_combined.txt", "a+")
    already_scraped = file.readlines()

    tot_rows = 0
    for f in name_files:
      # only combine files which we haven't already
      if f not in already_scraped:
        if f.startswith("dockets"):
          sub_df = pd.read_csv("../milestone2/dockets/" + f)
          tot_rows += sub_df.shape[0]
          all_dockets_df = all_dockets_df.append(sub_df)
          file.write(f)

        elif f.startswith("proceedings"):
          sub_df = pd.read_csv("../capstone/dockets/" + f)
          tot_rows += sub_df.shape[0]
          all_proceedings_df = all_proceedings_df.append(sub_df)
          file.write(f)


    all_dockets_df.to_csv('all_dockets.csv')
    all_proceedings_df.to_csv('all_proceedings.csv')

Part 3: Modeling¶

For our model, we want to predict bond given (likely) defendant race, charge, and magistrate section (aka commissioner/judge). In other words, we are creating a predictive model, where our features are race, charge, and mag. section, and our predicted variable is bond.

Again, we recognize the error in using ethnicolr to predict defendant race. Accordingly, we filter our race_charge DataFrame below to use only those defendants whose "most_likely_race" is above a certain threshold. For defendants who are most likely white, we set this threshold to be 90%; for all other defendants, we set this threshold to be 70%. We use a higher threshold for (likely) white defendants due to the over-representation of (likely) white people in our data set. There simply are not enough defendants who have a most_likely_race that is not white and most_likely_race_pct above 90%.

Out model is a KNearestNeighbors regression model for our predictions, with the number of neighbors set to 30. Given a new individual with feature variable values [f_0, f_1, ..., f_n], this model averages the values of the 30 points whose euclidean distance is closest to that new invididual as the individual's predicted value.

In [ ]:
# create a boolean mask for the conditions mentioned above
mask = ((race_charge["most_likely_race"] == "WHITE") & (race_charge["most_likely_race_pct"] >= 90)) | (
    (race_charge["most_likely_race"] != "WHITE") & (race_charge["most_likely_race_pct"] >= 70))
# apply that mask to the race_charge DataFrame to get a new, filtered DataFrame for our model                    
race_charge_filtered = race_charge[mask]

Function to output predicted bond given the features of our model. Note that we are only setting up the model once (above cell) and creating a function that allows us to call it as many times as we please, with whatever combination of features we desire.

In [ ]:
def predict_bond(charge, race, mag_sec):
    ''' given charge, race, and magistrate section, predict defendant's bond'''
    X_new_dict = [{
        "Charge": charge,
        "most_likely_race": race,
        "Mag Section": mag_sec,
    }]

    
    X_new = vec.transform(X_new_dict)
    
    X_new_sc = scaler.transform(X_new)
    return model.predict(X_new_sc)
